# Track 01 · DSA & Coding Interview

> **Goal:** Solve any LeetCode medium in 20 minutes and hard in 35. Build pattern recognition, not memorization.

---

## Why It Matters

Coding interviews at Google are pattern-matching tests disguised as algorithms problems. The gap between candidates is not intelligence — it is the breadth of patterns internalized and the discipline to decompose a problem cleanly under pressure. This track is the non-negotiable foundation.

---

## Core Knowledge

**Data Structures**
- Arrays, strings, hash maps, stacks, queues, heaps (min/max), deques
- Linked lists (fast/slow pointers, reversal, cycle detection)
- Trees: BST, AVL, segment tree, Fenwick tree, trie
- Graphs: adjacency list/matrix, union-find (DSU), topological sort

**Algorithm Patterns**
- Sliding window / two pointers
- Binary search (on answer, on rotated array, on condition)
- DFS / BFS (grid, graph, tree)
- Backtracking (permutations, combinations, constraint pruning)
- Dynamic programming (1D/2D, interval, knapsack, state machine)
- Monotonic stack / queue
- Divide and conquer / merge sort pattern
- Greedy with proof (exchange argument)

**Complexity Reasoning**
- Time and space complexity analysis for every solution
- Amortized analysis (dynamic arrays, hash map resizing)
- Space–time trade-off decisions

## How to Practice

1. **NeetCode 150** — complete the curated 150 as the baseline before expanding
2. **Pattern-first drilling** — group problems by pattern (e.g., 10 sliding window problems back to back), not by difficulty
3. **Timed sessions** — 25 minutes per problem, then analyze the optimal approach even if solved
4. **Blind solve after 1 week** — re-attempt problems you solved before without looking at your solution
5. **Mock interviews** — Pramp, interviewing.io, or with a partner, 2x per week from month 3 onward
6. **Verbalize** — explain your approach out loud as you code; silence kills in real interviews

**Weekly rhythm:** 5–7 problems/day minimum; 2 hard problems per week
